#    phi1 masters: (62,102,117)+(60,102,133) AND (102,102,117)+(100,102,133) ; settle ; remove
#    phi2 slaves : (46,102,117)+(44,102,133) AND (86,102,117)+(84,102,133)  ; settle ; remove
#
#  THE SHARED FAN-OUT BUS -- all four rails brought to dedicated lanes at z=152 (each >=2 cells apart:
#  x=39, 50, 79, 88), re-amped every <=9 dust. THE KEY RING-MAP INSIGHT: the west platform (x<=40) and the
#  x=67..80 gap and the east (x>=89) are ALL OPEN; each enclosed rail needs only to cross ONE ring wire:
#    * Q0  (x=44): tap @z=126, HOP OVER the x=41 forward-drop wire with a y=103 bridge, land x=39, lane

# =====================================================================================
# (A) bit0 physical self-advance + fan-out bus -- DIFFTEST-CLEAN at LOW  (reproduced on this jar)  OK
# =====================================================================================
# bit0's Qbar0->Master0.D T-loop is fully physical; on the shared two-phase clock Q0 (@44,101,122) toggles
# 0 -> 15 -> 0 -> 15 live (one metastability-break pulse first, then clean toggles -- build-12 behaviour).
# The fan-out bus reads the four rails' CORRECT live values across counter states:
#   state 00 (Q0=0): Q0BUS=0   Qbar0BUS=10  Q1BUS=0  Qbar1BUS=8   ({Q,Qbar}={0,1} both bits)
#   state 01 (Q0=1): Q0BUS=8   Qbar0BUS=0   Q1BUS=0  Qbar1BUS=8   (bit0 toggled; buses tracked it)
# (bus HIGH reads 8-10 = logical 1 attenuated over the ~30-cell lane; LOW reads 0.)
# bit0 STILL self-advances with the whole bus attached.
# difftest (console; verdict from log):
#   bit0 pure (no bus)              : difftest 44 101 122 200 -> BIT-IDENTICAL (200t, 432 cells, 218 comp)
#   bit0 + Q0 bridge (Q0=0)         : difftest 44 101 122 200 -> BIT-IDENTICAL (200t, 491 cells)
#   bit0 + Q0 & Qbar0 bus (Q0=0)    : difftest 44 101 122 200 -> BIT-IDENTICAL (200t, 505 cells)  [x2]
#   bit1 + Q1 & Qbar1 bus           : difftest 84 101 122 200 -> BIT-IDENTICAL (200t, 369 cells)  [x2]
# => the whole PC + the shared fan-out bus compiles BIT-FOR-BIT to vanilla when the bridged rails carry LOW,
#    and reads every rail's correct live value across states. The fan-out routing that build-17 said was the
#    fix is DELIVERED FUNCTIONALLY (all four rails out on their own lanes to a shared corridor, no crossing).
#
# =====================================================================================
# (B) NEW COMPILER FINDING -- vertical y=102/103 BRIDGE dust levels DIVERGE at HIGH  (difftest CAUGHT it)
# =====================================================================================
# The vertical bridge that carries an enclosed rail OVER the ring is difftest-clean when it carries LOW
# (real=sim=0, above), and an ISOLATED block-sourced bridge is fully clean:
#   vertical-parity spot-check (block source -> up -> over an independent wire -> down):
#     difftest 17 101 200 100 -> BIT-IDENTICAL (100t, 22 cells)   (VOUT=9)
# BUT when the ring-crossing bridge carries a HIGH bit (Q0=1, an attenuated/loaded dust level), the bridge's
# vertical-transition dust cells DIVERGE from vanilla by 1-2 levels -- and difftest CATCHES it:
#   bit0 + Q0 bridge, Q0=1, dust bridge       -> DIVERGENCE 800 mism; @tick0 (42,102,126) POWER real=9  sim=11
#   bit0 + Q0 bridge, Q0=1, repeater-sourced  -> DIVERGENCE 400 mism; @tick0 (40,102,126) POWER real=13 sim=14
# 400 mism / 200 ticks = 2 cells persistently off by 1 (the two climb/descent dust cells) -- a STEADY-STATE
# divergence, NOT a settle artifact (a full re-settle reproduced it). Re-sourcing the bridge from a clean
# repeater (strong-powering the climb) shrank the gap 2->1 but did NOT close it. So patch-0020's model of
# VERTICAL redstone-dust attenuation drifts 1-2 levels from vanilla when the bridge carries a nonzero loaded
# signal (it matches vanilla for the clean isolated case and for LOW). RECORDED as a compiler observation to
# feed back to patch 0020 (this map adds NO feature patch), exactly as build-06 (torch) and build-09
# (bistable-seeding) recorded compiler gaps the difftest discovered.
#
# CONSEQUENCE: the y=102 bridge is a FUNCTIONALLY-correct fan-out shortcut (values propagate, buses track the
# count) but it is NOT a reliably difftest-clean one. A fully difftest-BIT-IDENTICAL fan-out of the ENCLOSED
# rails needs a FLAT (single-layer, no vertical crossing) exit -- which requires re-laying bit0's forward +
# feedback interconnects on a genuinely WIDER footprint so a flat gap opens beside Q0/Qbar0 (build-17's
# "rebuild the ring bigger with deliberate gaps"). That flat re-lay is the remaining routing work; it was not
# completed in budget (it is a build-11/12-scale interconnect rebuild).
#
# =====================================================================================
# (C) XOR-gate CONSUME of the bus -> bit1.D  -- ATTEMPTED, layout bug found, not landed
# =====================================================================================
# Built the build-17 dual-rail XOR gate (bit1.D = max(Q1-Q0, Qbar1-Qbar0), two subtract comparators into a
# shared merge) as a CONSUMER fed by the four bus lanes (staggered inward turns so the west pair {Q0,Qbar0}
# and east pair {Q1,Qbar1} reach a central gate; one y=103 bridge to swap the pairing). It read WRONG
# (XV=15 at state 00 where XOR(0,0)=0). Root-caused by power reads:
#   1. the west-input runs cross the MERGE COLUMN (x=64, z=159..163): Q0's z=162 run passed through (64,162)/
#      (65,162) and Qbar0's z=158 run touched (64,158) adjacent to (64,159) -- both SHORT into the merge, so
#      the merge is pinned HIGH regardless of the comparators. (build-13/14 "merge must not be crossed by a
#      data run" lesson, re-learned.)
#   2. the long side-runs arrive ATTENUATED, so the subtract `15 - (attenuated Qbar0)` does not null (build-01
#      "a cleaning repeater must sit AT each comparator port" lesson).
# The CORRECT design (identified, not built in budget): route both comparator OUTPUTS away from the input
# rows to a merge that no data run crosses, and put a cleaning repeater AT every comparator rear AND side.
# The buggy gate was REMOVED (fill 38 101 153 90 103 172 air) to restore the difftest-clean fan-out bus as
# the artifact; both bits re-verified BIT-IDENTICAL (505 / 369 cells) after removal (bit0 at Q0=0).
#
