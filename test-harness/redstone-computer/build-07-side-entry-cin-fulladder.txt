# coderyoMC redstone-computer MAP — SIDE-ENTRY-Cin 1-bit FULL ADDER (SUM + Cout).
#
# The ripple-ready successor to build-03 (which was TOP-FED: every Cin port driven by a
# redstone_block at y+1, so a horizontal Cout->Cin carry could not enter — build-05 found
# the four Cin ports were buried with no horizontal access). THIS tile takes Cin as a
# HORIZONTAL dust/repeater BUS entering at the WEST edge (driver = redstone_block ABOVE
# the bus-start cell 4,20), so the Cout of bit i can be wired straight into the Cin bus of
# bit i+1 and the carry physically ripples.
#
#   SUM  = A XOR B XOR Cin
#   Cout = majority(A,B,Cin) = (A.B) + (A.Cin) + (B.Cin)
#
# CONVENTION (locked, empirically verified on this fork against build-01's NOT gate):
#   comparator/repeater [facing=D] => REAR/INPUT is on side D, OUTPUT exits the OPPOSITE
#   side; the two perpendicular cells are the comparator SIDES. SUBTRACT comparator:
#   out = max(0, rear - max(sideL, sideR)). A redstone_block ABOVE a dust drives it to 15.
#
# KEY DESIGN POINTS (each found the hard way; see RESULTS.md):
#  * XOR is built from a "merged-centre" |P-Q| pair whose two inputs alternate around the
#    perimeter (P,Q,P,Q) -> a planar single-layer crossover is impossible. SOLUTION: XOR2
#    uses TWO INDEPENDENT subtract comparators facing each other into ONE shared merge
#    cell (30,14); X1 is fed from the NW corner, Cin from the SE corner -> no crossover.
#  * Every comparator PORT is fed by a dedicated cleaning REPEATER placed right at the
#    port (so the analog Cin level can never corrupt a subtract). Cin is a repeater-
#    refreshed bus (a bare dust run dies after 15 blocks).
#  * The three Cout AND gadgets must be SPACED so no comp2's east SIDE touches the next
#    gadget's 15-source redstone_block (that poisons the subtract -> Cout stuck 0).
#  * The Cout OR-merge cleaning repeater must face SOUTH (rear=merge, out=Cout); facing
#    north silently drives the merge instead and Cout reads 0.
#
# VALIDATION (port 15568, -Dcoderyo.redstone.compile.enabled=true, flat world, force-
# loaded, mob-spawn + random-tick frozen by the difftest harness):
#   * SUM(30,101,14) + Cout(44,101,11) read correct = full-adder truth table for all 8
#     (A,B,Cin) combos (per-combo FRESH build, long settle — see the TPS note below).
#   * `/coderyo redstone difftest 30 101 14` = BIT-IDENTICAL to vanilla (the whole
#     connected network, ~270 cells), 0 divergences, every combo.
#
# !! SETTLE/TPS NOTE: this tile is a deep comparator cascade. On the regionized server
# the redstone load drops wall-clock TPS, so a probe must allow a LONG real-time settle
# (>=40 s) before reading — a 3-15 s settle reads a half-propagated state and looks like
# "Cin ignored". `difftest` is the reliable validator (it steps the model in GAME ticks,
# not wall-clock, so it is TPS-independent).
#
# Platform: y=100 stone, y=101 logic, y=102 input togglers (driver-placed).
# Boot then: forceload add 0 0 64 64 ; build (below) ; drive inputs ; settle ; difftest.
