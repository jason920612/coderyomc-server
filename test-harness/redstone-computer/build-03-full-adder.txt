# coderyoMC redstone-computer MAP — 1-bit FULL ADDER (SUM + Cout), VALIDATED.
#
# SUM  = A XOR B XOR Cin   (two cascaded XOR gadgets)
# Cout = majority(A,B,Cin) = (A AND B) OR (B AND Cin) OR (A AND Cin)   (3 ANDs + OR)
#
# STATUS: fully validated as a MAP. SUM 8/8 and Cout 8/8 input combos are
# BIT-IDENTICAL to vanilla under `/coderyo redstone difftest`, AND both truth
# tables are correct for all 8 (A,B,Cin) combinations. 16/16 difftests
# BIT-IDENTICAL, 0 divergences. (See RESULTS.md.)
#
# Convention (proven, same as build-01/02): comparator/repeater FACING points at
# the REAR/INPUT; output exits the opposite side. SUBTRACT comparator:
# out = max(0, rear - max(sideL, sideR)). Every comparator PORT is fed either a
# fresh 15 (redstone_block- or repeater-output-adjacent) or 0 — internal signals
# are repeater-cleaned right before each consumer so dust attenuation never
# corrupts the subtract arithmetic.
#
# BUGFIX vs the prior partial build-03: the X1 branch-2 distribution repeater at
# (32,101,88) was facing=north (rear pointing INTO the X1 bus, output dead-ended
# at the Pb port) so Pb never powered and SUM was wrong whenever X1=1 & Cin=1.
# It must be facing=south (rear=bus side, output=north toward Pb). With that one
# block corrected the whole SUM path settles and is BIT-IDENTICAL across all 8.
#
# Boot: -Dcoderyo.redstone.compile.enabled=true ; forceload the build area and
# settle (the prior agent's "false divergence" was an unticked chunk, not a
# compiler fault). Drive A,B,Cin with redstone_block togglers placed ONE BLOCK
# ABOVE (y=102) each input dust cell (see the toggler map at the bottom); the
# difftest re-reads those external sources live each tick.
#
# Platform: y=100 stone support, y=101 logic, y=102 input togglers (driver-placed).
