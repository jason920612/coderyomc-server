# The exact step that broke every prior build: tap Qbar0 (the FEEDBACK source) for a fan-out consumer.
# REPEATER (diode) tap on the feedback trunk: 52,101,146 repeater[facing=north] (rear on trunk (52,145)=Qbar0)
#   -> 52,147..150 wire -> 52,151 CLEANING repeater[facing=north] -> 52,152 wire (the consumer PORT).
# A repeater's rear only READS the trunk level; it sinks nothing and back-feeds nothing (one-way diode), so
# the feedback net is UNDISTURBED. Clocked 6+3 cycles with the tap attached:
#   Q0 @(44,101,122): 0,15,0,15,0,15,0,15,0 — STILL a clean toggle, tap did NOT break it.
#   fan-out bus mid @(52,101,150) = 12 (logical 1, re-amplified) / 0, tracking Qbar0 every cycle.
#   CLEANING-repeater PORT @(52,101,152) = 15 / 0 — a CLEAN 15, ready to feed an XOR subtract comparator.
#   difftest 44 101 122 200 (tap only)      -> BIT-IDENTICAL (200 ticks, 451 cells).
#   difftest 44 101 122 200 (tap + re-amp)  -> BIT-IDENTICAL (200 ticks, 454 cells).
# => the repeater-isolated fan-out KEEPS bit0 toggling cleanly AND is difftest BIT-IDENTICAL. The recurring
#    attenuation blocker is SOLVED by the repeater tap.

# (3) CONTROLLED CONTRAST — RAW-DUST tap at the SAME point ATTENUATES to nothing ✅  (why raw dust failed)
# =====================================================================================
# Single-variable swap: replace the repeater at (52,146) with RAW DUST, same route, extended 52,147..160:
#   Q0 still toggled here (this tap sits on the trunk, PAST the (48,131) feedback repeater that isolates the
#     D-path), BUT the fan-out is DEAD: BUSnear @(52,101,147) = 3 ; BUSfar @(52,101,160) = 0.
#   vs the REPEATER tap on the same net: bus = 12, port = 15.

#    repeater must sit AT every comparator port, fed by a repeater-isolated tap. (Historically, a raw tap
